Getting Started

by Douglas Clowes — last modified 2007-03-06 16:21

How to get onto the Instrument Control Computer and into the SICS software.

Introduction

If you are looking here it is probably because you need to do some work on the command line.  We have prepared a brief guide to useful system commands because the instrument control software runs on Linux computers with which many people may not be familiar.


There is an instrument control system computer (ICS) for each instrument. The ICS computer runs the SICS instrument control software which allows the computer to control the instrument. The SICS software uses a set of configuration files that define the controllable hardware of the instrument.

Users of the instrument must have a valid ICS username and password with which they can log into the ICS computer. This may not be the same as any other username and password that a user may have.

Users of the SICS instrument control software must have a valid SICS username and password with which they can log into the SICS software. This may not be the same as any other username and password that a user may have, and will usually be different.


Connecting to the ICS

You may use the ICS at the console of the computer, in the instrument cabin.

You may also connect to the ICS from the ANSTO network by using the ssh (secure shell) protocol. The name of each ICS computer is derived from the name of the instrument. The ICS computer for the Echidna instrument  is "ics1-echidna.nbi.ansto.gov.au".

In either case, you will be prompted to log into the computer.


Logging into the ICS

Once you are connected to the ICS, you should be prompted for your username. Log onto the ICS by entering your ICS username and password at the prompts.

When you have successfully logged into the ICS, you will be in your "home" directory.

Starting the SICS Software

The SICS software must be running on the instrument control computer. This may be started by calling the "runsics start"  command as shown below.

Note: the "echidna@ics1-echidna:~>"  is just the command line prompt.

echidna@ics1-echidna:~> runsics start

Starting SICS
29087
SUCCESS


Starting SICS Script Validator
29091
SUCCESS

You can also call "runsics status" for check if sics is running and "runsics stop" to stop SICS and the script validator.

Connecting to the SICS Software

From your home directory on the ICS computer, connect to SICS using the "socat" tool. This will give command line history and editing.

socat READLINE,history=echidna_history TCP4:localhost:server-echidna,crnl

In this example, the "socat" tool connects your keyboard and screen to the SICS software.

The READLINE method is used on the keyboard allowing command editing and history recall. The command history is saved between sessions in the file "echidna_history" in your current directory, which should be your home directory.

The connection is established by TCP/IP to the current computer ("localhost") on the port identified by the service "server-echidna" which is congigured into the system. On other instruments, replace the "echidna" with the name of the instrument.

Logging into the SICS Software

Once you are connected to SICS, you must log into SICS as a SICS user. There is no prompt for your username, only the generic "OK" prompt that SICS issues when asking for input.

Just type in your SICS username and password on the line and press the RETURN key. Your username and password will echo on the screen and be visible to any other people nearby.

If you are not successful, SICS will tell you. If you are successful, SICS will tell you that too.

OK
list
ERROR: Bad login
myusername mypassword
Login OK
In this example, the "OK" prompt was received after connecting to the SICS software. The "list" command was issued without logging in and SICS does not recognise it as a successful login. When a correct username and password is entered, SICS announces that the login was successful. SICS commands can now be entered.
